# The synthetic characters: two batches, both published

The substrate falsifier (Experiment 3) rates a set of synthetic, out-of-corpus
characters. Those characters were produced in two batches, and both are shipped
here. This note says what each batch is, which raters scored which, and why the
result does not depend on the difference.

## What happened

The synthetic characters were generated by Claude Opus 4.6 from a single prompt
asking for 20. The model returned **22** (`_run_synth.log`: "generating 20
synthetic characters ... wrote v0: 22 chars"). That 22-character batch, **batch
A**, is the one most of the panel scored. The set was then regenerated and
frozen at exactly 20, **batch B**, which is the version carried forward as the
released substrate. The two batches share no character names; they are
independent draws from the same design.

**A note on the name "v0", to disarm a collision.** The run log's "v0" refers to
the 22-character first draw (batch A). The shipped file named
`synthetic_characters_substrate_v0.json`, however, holds batch B's 20 characters,
byte-identical to `synthetic_characters_substrate.json`: the filename predates
the batch naming and was kept for compatibility with notebook 04. If you open
`_v0.json` expecting the 22-character first draw, the file you actually want is
`synthetic_characters_substrate_batchA_22.json`.

Both engineer the same structure the experiment relies on, an anti-correlation
between Honesty-Humility and Agreeableness in the designed targets:

| batch | file | characters | designed r(H, A) |
|---|---|---|---|
| A | `synthetic_characters_substrate_batchA_22.json` | 22 (Kessane, Sault, ...) | -0.391 |
| B (released) | `synthetic_characters_substrate.json` | 20 (Klaes, Trist, ...) | -0.744 |

## Which raters scored which

Of the 25 panel raters (`rater_substrate_map.csv` has the per-rater detail):

- **21** scored batch A (the 22-character set),
- **3** scored batch B (the released 20): DeepSeek-R1, Gemma-4-26B, Mistral-Large,
- **1** (DeepSeek-V3.1) was run across the regeneration and scored a set that
  spans both.

The headline numbers in `synthetic_vs_canonical.csv` (and therefore `panel25/`
and the paper's Figure 6) are computed on exactly what each rater saw. No rater's
score was recomputed to match one batch.

## Why the result does not depend on the batch

The off-cache collapse is unanimous within each batch, and if anything sharper on
the released batch B:

- batch A (21 raters): mean within-rater |r| falls from canonical to synthetic by
  about -0.43,
- batch B (the released 20, 3 raters): about -0.57 (and the one
  transition-spanning rater: -0.52).

Every one of the 25 raters shows the collapse, on whichever batch it scored. The
finding is a within-rater contrast between cached and uncached characters, so it
does not require the two batches to be identical, only that each is a valid
out-of-corpus set, which the cache-membership gauge confirms for both.

## Reproducing

- Both substrates are here with full evidence-pack text and designed targets.
- `rater_substrate_map.csv` maps each of the 25 raters to the batch it scored.
- To reproduce a single rater's synthetic |r|, use the substrate named for it in
  that map. To reproduce the released frozen set on its own, use
  `synthetic_characters_substrate.json` (batch B, 20 characters).

If you only want one clean, self-contained set to work with, use batch B: it is
the frozen 20 with the sharper designed anti-correlation, and it is the set the
practitioner tooling and the cache-map artifacts are built on.
